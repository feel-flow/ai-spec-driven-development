> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Tech Stack:** Markdown ドキュメント、Claude Code スラッシュコマンド、GitHub Copilot instructions

**Issue:** #344

```markdown
# ACE セットアップガイド

> このドキュメントは **AIツール（Claude Code, GitHub Copilot 等）が読んで、ユーザーと対話しながら ACE フレームワークをセットアップする** ためのガイドです。
>
> **概念説明**: [ACE_FRAMEWORK.md](./ACE_FRAMEWORK.md) を参照
>
> **クイックスタート**:
> - Claude Code: `/ace-setup` コマンドを実行
> - その他のAIツール: このドキュメントの URL を渡して「ACEをセットアップしてください」と指示
```

セクション構成:

1. **前提条件** — `/init-docs` 済み、`docs/` 存在が必要
2. **対話型セットアップフロー** — 5ステップの詳細手順（AIツールへの指示として構造化）
3. **テンプレート参照情報** — テンプレートリポジトリのパスとGitHub URL
4. **Copilot用ACE運用ルール テンプレート** — `.github/copilot-instructions.md` に追記する内容のテンプレート全文
5. **トラブルシューティング**

**対話フロー詳細（Step 1〜5）:**

Step 1（前提確認）では以下を自動チェックするようAIに指示:

- `docs/` と `docs/MASTER.md` の存在確認
  - `docs/` が存在しない場合: `/init-docs` の実行を推奨し、セットアップを中止する旨を明記
- PLAYBOOK.md の既存チェック（存在時は中止/バックアップの選択肢を提示）

Step 2（配置先の確認）:

- デフォルトパスを提示し、変更するか質問
- デフォルト: `docs/08-knowledge/PLAYBOOK.md`, `docs/05-operations/deployment/ace-cycle.md`

Step 3（ファイル配置）:

- テンプレートリポジトリの参照パスを明記
- ディレクトリ自動作成の指示
- PLAYBOOK.md テンプレートの重要ポイント: エントリ一覧は空にする（既存のサンプルエントリは削除）、Frontmatter の `owner` と `created`/`updated` をユーザーのプロジェクト情報で置換

Step 4（AIツール固有の設定）:

- 3つの選択肢（Claude Code / Copilot / 両方）
- Claude Code:
  - ace-curate.md のコピーとパス置換ルール（`docs-template/08-knowledge/PLAYBOOK.md` → Step 2 で確認したパス）
  - ace-curate.md が既に存在する場合はユーザーに確認: (a) スキップ / (b) 上書き — この旨を ACE_SETUP.md に明記すること
- Copilot:
  - 後述の「Copilot用ACE運用ルール テンプレート」を `.github/copilot-instructions.md` に追記
  - `.github/copilot-instructions.md` が存在しない場合は新規作成する旨を明記すること
  - 追記前に既存内容を確認し、ACE関連セクション（`## ACE` 等）が既にあればスキップする旨を明記すること

Step 5（完了確認）:

- 配置ファイル一覧の表示フォーマット
- ACEサイクル初回実行の案内
