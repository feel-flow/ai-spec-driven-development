# プロジェクト企画書テンプレート

## 📋 このドキュメントの目的

このテンプレートは、**何も決まっていない状態から、AIと一緒にプロジェクトを企画・立案する**ためのガイドです。

### 使い方
1. このテンプレートをコピーして、実際のプロジェクト名に変更
2. 各セクションを順番に埋めていく
3. AIツール（Claude Code、GitHub Copilot等）と対話しながら詳細化
4. 完成したら `PROJECT.md` と `MASTER.md` に情報を転記

### 関連ドキュメント

プロジェクトの特性に応じて、以下のテンプレートも活用してください：

- [INCEPTION_DECK_TEMPLATE.md](./INCEPTION_DECK_TEMPLATE.md) - アジャイル形式の10の質問でプロジェクトの方向性を明確化
- [USECASE_TEMPLATE.md](./USECASE_TEMPLATE.md) - 機能要件をアクター視点で詳細整理

## 8. 次のステップ

### 8.1 このテンプレートを埋めたら

1. **AIと対話してブラッシュアップ**

   ```
   Claude Codeなどに以下のように依頼：
   
   「この企画書をレビューして、以下を教えてください：
   - 曖昧な部分や詰めが甘い部分
   - 追加で検討すべき要件
   - 技術選定のアドバイス
   - リスク対策の提案」
   ```

2. **PROJECT.mdに転記**
   - ビジョン、ステークホルダー、要件定義などを `01-context/PROJECT.md` に整理

3. **CONSTRAINTS.mdに転記**
   - 技術的制約、予算、スケジュールなどを `01-context/CONSTRAINTS.md` に整理

4. **MASTER.mdの作成**
   - 技術スタックとコーディング規約を `MASTER.md` に記載

5. **ARCHITECTURE.mdの設計**
   - システム設計を `02-design/ARCHITECTURE.md` に記載

### 8.2 推奨される進め方

```
企画書（このファイル）
    ↓
PROJECT.md + CONSTRAINTS.md 作成
    ↓
技術選定 → MASTER.md 作成
    ↓
システム設計 → ARCHITECTURE.md 作成
    ↓
実装開始
```
