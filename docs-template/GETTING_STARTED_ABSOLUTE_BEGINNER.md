# 超入門ガイド - 何も決まっていない状態から始める

## 👋 このガイドの対象者

- **プログラミング初心者・未経験者**
- **「何か作りたいけど、何から始めればいいか分からない」という方**
- **AIツールの使い方が全く分からない方**
- **ドキュメント作成が初めての方**

**大丈夫です！このガイドに沿って進めれば、ゼロからプロジェクトを立ち上げられます。**

---

## 📖 全体の流れ（シンプル版）

```
STEP 1: 環境準備（1時間）
    ↓
STEP 2: アイデアを書き出す（30分）
    ↓
STEP 3: AIと対話して企画を作る（1時間）
    ↓
STEP 4: 技術を選ぶ（AIに相談）（1時間）
    ↓
STEP 5: 実装を始める準備（1時間）
    ↓
実装開始！
```

**合計所要時間: 約4.5時間**

---

## STEP 1: 環境準備（1時間）

### 1-1: 必要なツールのインストール

まず、以下のツールをインストールしましょう。

#### A. Visual Studio Code（VS Code）
無料のコードエディタです。

1. https://code.visualstudio.com/ にアクセス
2. 「Download」ボタンをクリック
3. ダウンロードしたファイルを開いてインストール
4. インストール完了後、VS Codeを起動

#### B. Git（バージョン管理）
コードの変更履歴を管理するツールです。

**macOSの場合:**
```bash
# ターミナルを開いて以下を実行
xcode-select --install
```

**Windowsの場合:**
1. https://git-scm.com/ にアクセス
2. 「Download」をクリック
3. インストーラーを実行（全て「Next」でOK）

**確認方法:**
```bash
git --version
# バージョンが表示されればOK
```

#### C. AIツール（どれか1つを選択）

**推奨: GitHub Copilot（初心者に優しい）**
- VS Code内で使える
- 月額$10（個人）
- 初月無料トライアルあり

**または: Claude Code（高機能）**
- 高度なコード生成が可能
- 無料プラン/有料プランあり

**または: Cursor（VS Codeベース）**
- VS Codeと似た操作感
- AI機能が統合されている

**👉 詳細なセットアップ方法:**
- GitHub Copilot: [`SETUP_GITHUB_COPILOT.md`](./SETUP_GITHUB_COPILOT.md)
- Claude Code: [`SETUP_CLAUDE_CODE.md`](./SETUP_CLAUDE_CODE.md)
- Cursor: [`SETUP_CURSOR.md`](./SETUP_CURSOR.md)

### 1-2: プロジェクトフォルダの作成

```bash
# ターミナル（macOS）またはコマンドプロンプト（Windows）で実行

# 1. プロジェクト用のフォルダを作成
mkdir my-first-project

# 2. フォルダに移動
cd my-first-project

# 3. Gitを初期化
git init

# 4. VS Codeで開く
code .
```

**✅ STEP 1 完了チェック:**
- [ ] VS Codeがインストールされている
- [ ] Gitがインストールされている
- [ ] AIツールが使える状態になっている
- [ ] プロジェクトフォルダを作成できた

## STEP 2: アイデアを書き出す（30分）

### 2-1: 企画書テンプレートをコピー

まず、このリポジトリをクローン（コピー）します。

```bash
# プロジェクトフォルダとは別の場所で実行
git clone https://github.com/feel-flow/ai-spec-driven-development.git
```

次に、企画書テンプレートをコピーします。

```bash
# your-project フォルダに移動
cd my-first-project

# docsフォルダを作成
mkdir -p docs/00-planning

# テンプレートをコピー
cp ../ai-spec-driven-development/docs-template/00-planning/PLANNING_TEMPLATE.md \
   docs/00-planning/my-planning.md
```

### 2-2: 企画書を開いて書き始める

VS Codeで `docs/00-planning/my-planning.md` を開きます。

**まず、以下の3つの質問に答えてください:**

#### Q1: 何を作りたいですか？（1行で）

```
例: 毎日の習慣を記録・管理できるアプリ
例: レシピを保存・検索できるWebサイト
例: 家計簿アプリ
```

**あなたの答え:**
```
[ここに記入]
```

#### Q2: なぜ作りたいのですか？（困っていること）

```
例: 既存の習慣管理アプリは機能が多すぎて使いにくい
例: レシピサイトは広告が多くて見づらい
例: シンプルな家計簿アプリが見つからない
```

**あなたの答え:**
```
[ここに記入]
```

#### Q3: 誰が使いますか？

```
例: 自分自身
例: 家族
例: 友人
例: 一般の人（不特定多数）
```

**あなたの答え:**
```
[ここに記入]
```

### 2-3: 最初の機能を考える（3つだけ）

**絶対に必要な機能を3つだけ書いてください:**

```
例（習慣管理アプリの場合）:
1. 習慣を登録できる
2. 毎日チェックできる
3. 達成状況をグラフで見られる
```

**あなたの答え:**
```
1. [機能1]
2. [機能2]
3. [機能3]
```

**✅ STEP 2 完了チェック:**
- [ ] 企画書テンプレートをコピーできた
- [ ] Q1-Q3に答えた
- [ ] 必要な機能を3つ考えた

---

## STEP 3: AIと対話して企画を作る（1時間）

### 3-1: AIに企画をレビューしてもらう

あなたが選んだAIツール（GitHub Copilot / Claude Code / Cursor）で、以下のプロンプトを使います。

**コピー＆ペーストしてください:**

```
以下のプロジェクト企画をレビューして、アドバイスをください。

【作りたいもの】
[STEP 2のQ1をコピー]

【理由】
[STEP 2のQ2をコピー]

【ユーザー】
[STEP 2のQ3をコピー]

【機能】
[STEP 2の機能3つをコピー]

以下を教えてください：
1. このアイデアは実現可能ですか？
2. 似たようなサービス・アプリはありますか？
3. 追加で必要そうな機能はありますか？
4. 初心者が作るには難易度はどれくらいですか？
5. どの技術を使うのがおすすめですか？

分かりやすく、優しく説明してください。
```

### 3-2: AIの回答を読んで、企画書を更新

AIからの回答を読んで、以下を企画書（`my-planning.md`）に追記します：

- **セクション3.1 コア機能（MVP）**: 機能を詳しく書く
- **セクション4.1 技術スタックの検討**: AIが提案した技術を記入
- **セクション6.1 想定されるリスク**: AIが指摘したリスクを記入

### 3-3: もう一度AIに相談

企画書を更新したら、もう一度AIに確認します。

```
企画書を更新しました。以下の内容で問題ないか確認してください。

[更新した企画書の内容をコピー＆ペースト]

特に以下を確認してください：
1. 機能が多すぎませんか？（初心者なので）
2. 技術選定は適切ですか？
3. 最初に何から作り始めるべきですか？
```

**✅ STEP 3 完了チェック:**
- [ ] AIに企画をレビューしてもらった
- [ ] 企画書を更新した
- [ ] AIから「問題ない」という回答をもらった

---

## STEP 4: 技術を選ぶ（AIに相談）（1時間）

### 4-1: 技術スタックを決める

AIに以下のプロンプトで相談します。

```
初心者が [あなたのプロジェクト] を作る場合、
以下の観点で技術スタックを提案してください。

【重視する点】
1. 学習が簡単（初心者向け）
2. 無料で使える
3. 情報（チュートリアル等）が多い
4. 将来的に拡張しやすい

【必要な技術】
- フロントエンド（見た目）
- バックエンド（サーバー側）※必要な場合
- データベース（データ保存）※必要な場合
- ホスティング（公開方法）

それぞれの技術について：
- 技術名
- 選んだ理由
- 学習リソース（チュートリアルURL等）

を教えてください。
```

### 4-2: MASTER.mdを作成

AIが提案した技術スタックを、MASTER.mdに記録します。

```bash
# テンプレートをコピー
cp ../ai-spec-driven-development/docs-template/MASTER.md \
   docs/MASTER.md
```

VS Codeで `docs/MASTER.md` を開いて、以下を記入：

1. **プロジェクト識別情報**
   - プロジェクト名: [あなたのプロジェクト名]
   - バージョン: 0.1.0
   - 最終更新日: [今日の日付]

2. **プロジェクト概要**
   - 何を作るか: [STEP 2のQ1]
   - なぜ作るか: [STEP 2のQ2]
   - 誰のためか: [STEP 2のQ3]

3. **技術スタック**
   - [AIが提案した技術を記入]

**✅ STEP 4 完了チェック:**
- [ ] AIに技術スタックを提案してもらった
- [ ] MASTER.mdを作成した
- [ ] 技術スタックを記入した

## STEP 5: 実装を始める準備（1時間）

### 5-1: 開発環境をセットアップ

AIに開発環境のセットアップを依頼します。

```
以下の技術スタックで開発環境をセットアップする手順を、
初心者向けに分かりやすく教えてください。

【技術スタック】
[MASTER.mdの技術スタックをコピー]

【知りたいこと】
1. インストールするソフトウェア（とそのコマンド）
2. プロジェクトの初期化コマンド
3. 必要なライブラリのインストールコマンド
4. 動作確認の方法

ステップバイステップで、コピー＆ペーストできるコマンドで教えてください。
```

### 5-2: AIの指示に従ってセットアップ

AIが教えてくれたコマンドを、順番に実行していきます。

**💡 ヒント:**
- コマンドは1行ずつ実行
- エラーが出たら、エラーメッセージをAIに見せて相談
- 分からない用語があったら、AIに質問

### 5-3: 最初の「Hello World」を作る

開発環境が整ったら、最初の小さなプログラムを作ります。

```
開発環境のセットアップが完了しました。

【確認したいこと】
動作確認として、画面に「Hello World」と表示する
一番シンプルなプログラムを作りたいです。

以下を教えてください：
1. どのファイルにコードを書けばいいですか？
2. コード全体（コピー＆ペーストできる形）
3. 実行方法
4. 確認方法（どうなれば成功？）

初心者なので、詳しく説明してください。
```

### 5-4: 動作確認

AIが教えてくれた方法で、「Hello World」が表示されることを確認します。

**成功したら、おめでとうございます！🎉**

**✅ STEP 5 完了チェック:**
- [ ] 開発環境をセットアップできた
- [ ] Hello Worldを表示できた
- [ ] 基本的な実行方法が分かった

## 🎉 実装開始！

ここまで完了すれば、いよいよ実際の機能を作り始められます！

### 最初の機能を作る

企画書で決めた「機能1」を作りましょう。

**AIへのプロンプト:**

```
【タスク】
[企画書の機能1をコピー]

【前提】
- 初心者なので、できるだけシンプルに
- コメントを多めに書いて
- どのファイルに書くか明示して

【成果物】
1. コード全体（コピー＆ペーストできる形）
2. 実行方法
3. 動作確認の方法

一つずつ丁寧に説明してください。
```

### 作業の進め方

```
1. AIにコードを生成してもらう
   ↓
2. コピー＆ペーストして保存
   ↓
3. 実行して動作確認
   ↓
4. エラーが出たら、AIに相談
   ↓
5. 動いたら次の機能へ
   ↓
繰り返し
```

## 😰 困ったときは

### エラーが出た場合

1. **エラーメッセージをコピー**
2. **AIに以下のように相談:**

```
以下のエラーが出ました。どうすれば直せますか？

【エラーメッセージ】
[エラーメッセージをコピー]

【やったこと】
[実行したコマンドや操作]

初心者なので、分かりやすく説明してください。
```

### 用語が分からない場合

```
「[分からない用語]」とは何ですか？
初心者向けに、簡単な言葉で説明してください。
```

### もっと詳しく知りたい場合

```
[知りたいこと] について、もっと詳しく教えてください。

具体例や図があると分かりやすいです。
```
