# 新規プロジェクト完全ガイド - ゼロから始めるAI仕様駆動開発

## 🎯 このガイドの対象者

- **完全に新しいプロジェクトを始める方**
- **何を作るか、まだ決まっていない段階の方**
- **AIツールと一緒にプロジェクトを企画・設計したい方**

---

## 📖 全体の流れ（所要時間: 約8-12時間）

```
Phase 0: 企画・アイデア出し (2-3時間)
    ↓
Phase 1: 要件定義 (2-3時間)
    ↓
Phase 2: 技術選定 (1-2時間)
    ↓
Phase 3: システム設計 (2-3時間)
    ↓
Phase 4: 実装準備 (1時間)
    ↓
実装開始！
```

---

## Phase 0: 企画・アイデア出し（2-3時間）

### ステップ0-1: 企画書テンプレートを開く

1. **企画書をコピー**

   ```bash
   cp docs-template/00-planning/PLANNING_TEMPLATE.md \
      docs/00-planning/my-project-planning.md
   ```

2. **企画書を開く**
   - `docs/00-planning/my-project-planning.md` を開く

### ステップ0-2: アイデアを書き出す

企画書の**セクション1〜3**を埋めていきます：

1. **きっかけと解決したい課題**
   - なぜこのプロジェクトを始めるのか？
   - どんな問題を解決したいのか？

2. **ターゲットユーザー**
   - 誰が使うのか？
   - どんな価値を提供するのか？

3. **コア機能（MVP）**
   - 最初のリリースで絶対に必要な機能3-5個

### ステップ0-3: AIにレビューを依頼

**Claude Code / GitHub Copilot に以下のプロンプトを投げる：**

```
以下のプロジェクト企画をレビューしてください。

【企画内容】
[企画書の内容をコピー＆ペースト]

【レビュー観点】
1. 曖昧な部分や詰めが甘い部分を指摘
2. 追加で検討すべき要件を提案
3. 類似サービス・競合の分析
4. 実現可能性の評価（技術的・リソース的）
5. リスクの洗い出しと対策案

具体的で実行可能なフィードバックをお願いします。
```

### ステップ0-4: フィードバックを反映

AIからのフィードバックを元に企画書をブラッシュアップします。

**チェックリスト:**
- [ ] ビジョンが30秒で説明できる
- [ ] ターゲットユーザーが明確
- [ ] MVP機能が3-5個に絞られている
- [ ] 成功指標（KPI）が定義されている
- [ ] 主要なリスクが洗い出されている

### ステップ1-2: PROJECT.mdを作成

1. **テンプレートをコピー**

   ```bash
   cp docs-template/01-context/PROJECT.md docs/01-context/PROJECT.md
   ```

2. **企画書の内容を転記**

   | 企画書のセクション | PROJECT.mdのセクション |
   |-------------------|---------------------|
   | 1. 発想・アイデア | 2. プロジェクトビジョン |
   | 2. ビジョンと目標 | 2. プロジェクトビジョン |
   | 3. 機能要件 | 4. 要件定義 > 機能要件 |
   | 7. ステークホルダー | 3. ステークホルダー分析 |

3. **AIに詳細化を依頼**

```
以下のPROJECT.mdを、より具体的で実行可能な内容に詳細化してください。

🚨 重要: 情報が不足している場合は推論せず、必ず確認してください。

【現在の内容】
[PROJECT.mdの内容をコピー]

【詳細化してほしい点】
1. 機能要件をユーザーストーリー形式で記述
   - As a [ユーザー], I want to [機能], so that [目的]
2. 非機能要件の具体化
   - パフォーマンス、セキュリティ、可用性
3. 受け入れ基準の明確化
   - Given/When/Thenで記述

【確認ルール】
以下の情報が不明な場合は、推論で進めず確認してください：
- ターゲットユーザーの具体的なペルソナ
- パフォーマンス要件の数値（応答時間、同時接続数等）
- セキュリティレベル（個人情報の扱い、認証方式等）
- 対応デバイス・ブラウザの範囲

不明な点があれば、以下の形式で質問してください：
```

⚠️ 情報不足により確認が必要です

【必須確認事項】
- 理由: [なぜ確認が必要か]
- 推奨: [推奨される選択肢]

```
```

## Phase 2: 技術選定（1-2時間）

### ステップ2-1: 技術スタックの選定

**AIに技術選定を依頼:**

```
以下の要件と制約を満たす技術スタックを提案してください。

【要件】
[PROJECT.mdの機能要件・非機能要件をコピー]

【制約】
[CONSTRAINTS.mdの技術的制約をコピー]

【重視する点】
1. 学習コスト（チームのスキルレベル: [初級/中級/上級]）
2. 開発スピード（リリースまでの期限: [期限]）
3. 運用コスト（予算: [予算]）
4. スケーラビリティ（想定ユーザー数: [数]）
5. エコシステムの充実度

以下の形式で提案してください：
- フロントエンド: [技術] - [選定理由]
- バックエンド: [技術] - [選定理由]
- データベース: [技術] - [選定理由]
- インフラ: [技術] - [選定理由]
- 開発ツール: [技術] - [選定理由]
```

### ステップ2-2: MASTER.mdを作成

1. **テンプレートをコピー**

   ```bash
   cp docs-template/MASTER.md docs/MASTER.md
   ```

2. **プロジェクト情報を記入**

   ```markdown
   ## プロジェクト識別情報
   - **プロジェクト名**: [企画書から]
   - **バージョン**: 0.1.0
   - **使用AIツール**: Claude Code, GitHub Copilot, Cursor
   - **最終更新日**: [今日の日付]

   ## プロジェクト概要
   - **何を作るか**: [企画書のビジョンから]
   - **なぜ作るか**: [企画書の課題から]
   - **誰のためか**: [企画書のターゲットから]

   ## 技術スタック
   [ステップ2-1で選定した技術スタックを記入]
   ```

3. **コーディング規約を設定**

   MASTER.mdには、以下のコーディングルールがテンプレートとして含まれています：
   - 型安全性（TypeScriptなど）
   - エラーハンドリング方針
   - テストカバレッジ目標
   - 命名規則
   - マジックナンバー禁止

   **プロジェクトに合わせてカスタマイズしてください。**

**チェックリスト:**
- [ ] 技術スタックが決定し、MASTER.mdに記載
- [ ] 選定理由が明確
- [ ] コーディング規約が設定されている
- [ ] AIツール連携の準備完了

### ステップ4-1: PATTERNS.mdを作成


### ステップ4-2: TESTING.mdを作成

1. **テンプレートをコピー**

   ```bash
   cp docs-template/04-quality/TESTING.md docs/04-quality/TESTING.md
   ```

2. **テスト戦略を定義**
   - Unit Test: 80%以上のカバレッジ
   - Integration Test: 主要なフローをカバー
   - E2E Test: クリティカルパスのみ

### ステップ4-3: 開発環境のセットアップ

**AIに初期プロジェクト構築を依頼:**

```
以下の技術スタックで新規プロジェクトを初期化するコマンドを教えてください。

【技術スタック】
[MASTER.mdの技術スタックをコピー]

【含めてほしいもの】
- プロジェクト初期化コマンド
- 必要なライブラリのインストールコマンド
- 基本的なディレクトリ構造（コードとドキュメントの両方）
  - コード: src/, tests/, config/ 等
  - ドキュメント: docs-template/ の構造（`数字-英語小文字`ディレクトリ + `大文字.md`ファイル）
  - 詳細な命名規則は `docs-template/03-implementation/CONVENTIONS.md` を参照
- 設定ファイル（tsconfig.json, .eslintrc等）
- GitHubリポジトリ作成手順

```

## 🎉 実装開始

ここまで完了すれば、AI駆動開発の準備が整いました！

### 開発サイクル

```
1. PROJECT.mdから次の機能を選ぶ
   ↓
2. AIに実装を依頼（MASTER.md、PATTERNS.mdに従う）
   ↓
3. 生成されたコードをレビュー
   ↓
4. テストを実行して確認
   ↓
5. 必要に応じて調整
   ↓
6. コミット＆プッシュ
   ↓
繰り返し
```

### AIへの実装依頼テンプレート

```
【タスク】
[PROJECT.mdの機能をコピー]

【前提条件】
- docs/MASTER.md のルールに従う
- docs/03-implementation/PATTERNS.md のパターンを使用
- docs/02-design/ARCHITECTURE.md の設計に沿う

【制約】
- マジックナンバー禁止
- any型禁止
- エラーハンドリング必須
- テスト同時生成
- 関数は30行以内

【成果物】
1. 実装コード
2. ユニットテスト（80%以上カバレッジ）
3. 型定義
4. 必要なコメント

実装してください。
```
