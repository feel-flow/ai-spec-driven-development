# 整合性校閲コマンド

引数で指定されたファイル（複数章推奨）の整合性を検証し、章間の矛盾や用語定義のブレを検出してください。

## 実行手順

1. `.claude/skills/proofread-consistency.md` のスキル内容を読み込む
2. 指定されたファイル群をスキャン
3. 用語の初出と定義を追跡
4. 章間の参照（「前章」「後述」）を確認
5. 整合性を検証してレポート

## 検証対象

- 用語定義の一貫性（同一用語に異なる定義がないか）
- 章間の数値の不一致（「7文書」→「6文書」など）
- クロスリファレンスの正確性（「前章で述べた」が存在するか）
- 概念の進化と整合（導入→詳細→応用の流れ）
- 暗黙の前提変化（対象読者、スコープの変化）

## 推奨使用方法

このコマンドは書籍全体を対象にすることを推奨します。

## 引数

- `$ARGUMENTS` - 検証対象のディレクトリパス（書籍全体推奨）

## 使用例

```
# 書籍全体の整合性チェック（推奨）
/proofread-consistency books/ai-spec-driven-development-90percent/

# 特定パートのみ
/proofread-consistency books/ai-spec-driven-development-90percent/part3_practice/
```
