# 主張校閲コマンド

引数で指定されたファイル内の主張を検証し、根拠の有無、断言の妥当性を検出してください。

## 実行手順

1. `.claude/skills/proofread-claims.md` のスキル内容を読み込む
2. 指定されたファイルをスキャン
3. 断言・評価・推奨表現を抽出
4. 根拠の有無と妥当性を検証
5. 結果をレポート

## 検証対象

- 根拠のない断言（「〜である」「〜だ」）
- 曖昧な伝聞（「〜と言われている」）
- 権威への訴え（「専門家によると」）
- 過度な断言（「絶対に」「必ず」「常に」）
- 過度な一般化（個人の経験→全員に適用）
- 主観と客観の混同

## 引数

- `$ARGUMENTS` - 検証対象のファイルパスまたはディレクトリ

## 使用例

```
/proofread-claims books/ai-spec-driven-development-90percent/part2_spec-is-90percent/
```
