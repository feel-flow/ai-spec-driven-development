# 論理校閲スキル

## 役割

文章内の論理構造を検証し、矛盾や論理の飛躍を検出します。

**対象**: 技術書、ビジネス文書、論説文

## 検証対象

### 1. 論理の一貫性

#### 検証パターン

| パターン | 例 | 問題点 |
| --- | --- | --- |
| 自己矛盾 | 「Aは必須」→「Aは不要」 | 同一文書内での矛盾 |
| 前提と結論の不一致 | 「Xが原因」→「Yを解決」 | 因果関係の断絶 |
| 条件の欠落 | 「常にAである」 | 例外条件の無視 |

#### 検証方法

1. **主張の抽出**: 「〜である」「〜すべき」「〜は必須」などの断定表現を収集
2. **関係性の分析**: 主張間の論理関係（因果、対立、補完）を分析
3. **矛盾の検出**: 相反する主張がないか確認

### 2. 因果関係の妥当性

#### 検証項目

| 項目 | 説明 | 例 |
| --- | --- | --- |
| 因果の飛躍 | A→B→C が A→C と省略 | 「AIを使えば生産性が上がる」 |
| 逆因果 | 原因と結果が逆転 | 「成功したから準備した」 |
| 相関と因果の混同 | 相関を因果として主張 | 「Xが多いとYも多い→XがYの原因」 |
| 隠れた前提 | 暗黙の前提が存在 | 「これを使えば解決」（条件不明） |

#### 検証方法

```
因果関係のチェックリスト:
□ 原因と結果が明確に示されているか
□ 因果の方向は正しいか
□ 中間のステップが省略されていないか
□ 他の要因の可能性は考慮されているか
□ 前提条件が明示されているか
```

### 3. 論理の飛躍

#### 危険なパターン

**🔴 重大（修正必須）**

```
パターン: 根拠なき一般化
例: 「私の経験では成功した」→「誰でも成功する」
問題: 個別事例から一般論への飛躍
```

```
パターン: 二項対立の強制
例: 「Aでなければ失敗」
問題: 中間的な選択肢の無視
```

**🟡 警告（確認推奨）**

```
パターン: 暗黙の前提
例: 「当然ながら〜」「言うまでもなく〜」
問題: 読者が前提を共有しているとは限らない
```

### 4. 推論の妥当性

#### 検証すべき推論パターン

| パターン | 説明 | 検証ポイント |
| --- | --- | --- |
| 演繹的推論 | 前提→結論 | 前提が正しく、推論が妥当か |
| 帰納的推論 | 複数の事例→一般化 | 事例が十分か、例外はないか |
| 類推 | A:B = C:D | 類似性は本質的か |

#### 検証方法

1. 前提の確認: 主張の根拠となる前提を抽出
2. 推論過程の確認: 前提から結論への論理ステップを検証
3. 結論の妥当性: 結論が前提から正当に導かれているか

### 5. 条件と例外

#### 検証項目

| 項目 | 問題パターン | 推奨修正 |
| --- | --- | --- |
| 絶対表現 | 「常に」「絶対に」「必ず」 | 条件や例外を明記 |
| 普遍主張 | 「すべてのXはY」 | 範囲を限定 |
| 無条件推奨 | 「これを使えばよい」 | 前提条件を明示 |

## 検証プロセス

### ステップ1: 主張の抽出

テキストから以下を抽出:

1. 断定表現: 「〜である」「〜だ」
2. 推奨表現: 「〜すべき」「〜が良い」
3. 因果表現: 「〜だから」「〜のため」「〜すると」
4. 条件表現: 「〜の場合」「〜であれば」

### ステップ2: 論理構造の可視化

```
主張1: [内容]
  ├─ 根拠: [根拠1], [根拠2]
  ├─ 前提: [暗黙の前提]
  └─ 関連: 主張3と矛盾の可能性

主張2: [内容]
  ├─ 根拠: [根拠]
  └─ 結論: 主張4の前提
```

### ステップ3: 矛盾・飛躍の検出

各主張について:
1. 他の主張と矛盾していないか
2. 根拠から結論への飛躍がないか
3. 前提条件は明示されているか

## チェックリスト

検証時に確認すべき項目:

- [ ] 同一文書内で矛盾する主張がないか
- [ ] 因果関係に飛躍がないか
- [ ] 絶対表現（常に、必ず、すべて）に例外の考慮があるか
- [ ] 「当然」「言うまでもなく」などの暗黙の前提がないか
- [ ] 結論が前提から妥当に導かれているか
- [ ] 相関と因果を混同していないか
