# 日本語校正スキル

## 役割

日本語の文章品質を検査し、誤字脱字、文法の問題、読みやすさの改善点を検出します。

## 確認観点

### 1. 誤字・脱字

#### 漢字の誤変換

よくある誤変換パターン：

| 正 | 誤（よくある間違い） |
| --- | --- |
| 仕様 | 私用、使用（文脈による） |
| 実装 | 実相、実蔵 |
| 精度 | 制度、性度 |
| 生成 | 成生、製成 |
| 認証 | 認識（文脈による） |
| 環境 | 還境、関境 |
| 変換 | 返還、変管 |

#### 送り仮名の揺れ

統一すべきパターン：

| 推奨 | 許容 | 備考 |
| --- | --- | --- |
| 行う | 行なう | 「行う」を推奨 |
| 表す | 表わす | 「表す」を推奨 |
| 現れる | 現われる | 「現れる」を推奨 |
| 捉える | 捕らえる | 意味が異なる場合あり |

#### 同音異義語

注意が必要な組み合わせ：

- 以上 / 異常 / 移譲
- 意志 / 意思
- 回答 / 解答
- 機能 / 帰納
- 精度 / 制度
- 対象 / 対称 / 対照

### 2. 文法チェック

#### 主語と述語の対応

```
🔴 悪い例：
「この機能は、ユーザーが設定画面から変更できます。」
→ 主語「機能は」と述語「変更できます」が対応していない

✅ 良い例：
「ユーザーは、この機能を設定画面から変更できます。」
「この機能は、設定画面から変更できます。」
```

#### 助詞の誤用

よくある誤用パターン：

- 「〜を行う」→「〜を実行する」（重複表現）
- 「〜において」の多用 →「〜で」に簡略化を検討
- 「〜に対して」vs「〜について」の混同

#### 敬体と常体の混在

```
🔴 悪い例：
「この章では、AIの限界を解説します。実際、AIは完璧ではない。」
→ 「します」（敬体）と「ではない」（常体）が混在

✅ 良い例：
「この章では、AIの限界を解説します。実際、AIは完璧ではありません。」
```

**例外**：引用やコード例内は常体でも可

### 3. 読みやすさ

#### 文の長さ

| 基準 | 目安 |
| --- | --- |
| 推奨 | 60文字以内 |
| 許容 | 80文字以内 |
| 要分割 | 80文字超 |

```
🔴 悪い例（95文字）：
「AIに大きなタスクを丸投げすると、だいたい70%くらいの完成度で止まってしまうという現象は、ChatGPT、Claude、画像生成AIなど、すべてのAIツールで共通して発生しています。」

✅ 良い例（分割後）：
「AIに大きなタスクを丸投げすると、だいたい70%くらいの完成度で止まってしまいます。
この現象は、ChatGPT、Claude、画像生成AIなど、すべてのAIツールで共通して発生しています。」
```

#### 段落の長さ

- 推奨：1段落あたり3〜5文
- 要分割：6文以上の段落

#### 修飾語の係り受け

```
🔴 曖昧な例：
「高速に動作するAIが生成したコード」
→ 「高速に動作する」は「AI」にかかるのか「コード」にかかるのか不明

✅ 明確な例：
「AIが高速に動作して生成したコード」（AIが高速）
「AIが生成した、高速に動作するコード」（コードが高速）
```

### 4. 技術書特有の注意点

#### 冗長表現

| 冗長 | 簡潔 |
| --- | --- |
| することができます | できます |
| することが可能です | できます |
| を行います | します |
| というものです | です |
| といったような | のような |
| 〜という | 〜（削除可能な場合） |

#### 「〜になります」の誤用

```
🔴 誤用：
「結果はこちらになります。」（変化を伴わない）

✅ 正しい使い方：
「設定を変更すると、UIがダークモードになります。」（変化を伴う）
「結果はこちらです。」（変化を伴わない場合）
```

#### 二重否定の回避

```
🔴 悪い例：
「できないわけではありません」
「問題がないとは言えません」

✅ 良い例：
「可能です」
「問題がある可能性があります」
```

#### カタカナ語の過度な使用

技術用語は必要ですが、日本語で表現できる場合は検討：

| カタカナ | 日本語代替（検討） |
| --- | --- |
| ソリューション | 解決策 |
| イシュー | 課題、問題 |
| アジェンダ | 議題 |
| コンセンサス | 合意 |

**注意**：技術的な文脈では原語を使う方が適切な場合も多い

## 出力形式

```markdown
### 日本語校正結果

#### 🔴 誤字・脱字
- **[行XX]** 「[誤]」→「[正]」
  - 理由: [誤変換/送り仮名/同音異義語]

#### 🟡 文法の問題
- **[行XX]** 問題: [説明]
  - 現在: 「[元の文]」
  - 修正案: 「[改善案]」

#### 🔵 読みやすさの改善
- **[行XX]** 長文（[N文字]）
  - 現在: 「[元の文]」
  - 提案: 「[分割案]」

- **[行XX]** 冗長表現
  - 現在: 「することができます」
  - 提案: 「できます」
```

## チェックリスト

校正時に確認すべき項目：

- [ ] 漢字の誤変換がないか
- [ ] 送り仮名は統一されているか
- [ ] 同音異義語の誤用がないか
- [ ] 主語と述語は対応しているか
- [ ] 敬体・常体が混在していないか
- [ ] 80文字を超える文がないか
- [ ] 冗長表現を簡潔にできないか
- [ ] 二重否定を避けているか
