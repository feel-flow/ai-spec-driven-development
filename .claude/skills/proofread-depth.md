# 内容充実度検証スキル（/proofread-depth）

## 概要

章の「内容の深さ・充実度」をチェックします。構造や日本語の正確さではなく、**読者が本当に学べる内容になっているか**を評価します。

## 対象書籍

`books/ai-small-is-accurate/` 配下の章ファイル

## チェック項目

### 1. 具体例の充実度 🔍

各主要セクションに具体例があるかを検証します。

**チェックポイント**:
- [ ] 各主要セクション（H2レベル）に具体例があるか
- [ ] Before/Afterの例では「なぜ改善したか」の説明があるか
- [ ] 依頼→出力→問題点→結果の流れが示されているか
- [ ] 抽象的な説明だけで終わっていないか

**評価基準**:

| 評価 | 基準 |
| --- | --- |
| 🟢 優秀 | 全主要セクションに具体例があり、因果関係も明確 |
| 🟡 要改善 | 一部セクションに具体例がない、または説明が不足 |
| 🔴 要追加 | 具体例がほとんどない、抽象論のみ |

### 2. ケーススタディ 📋

実際のシナリオに基づく事例があるかを検証します。

**チェックポイント**:
- [ ] 抽象論だけでなく実際のシナリオがあるか
- [ ] 背景→依頼→AIの出力→問題点→結果の構造になっているか
- [ ] 具体的な数字（時間、コメント数、修正回数など）があるか
- [ ] 読者が「自分もこうなりそう」と思える共感ポイントがあるか

**評価基準**:

| 評価 | 基準 |
| --- | --- |
| 🟢 優秀 | 詳細なケーススタディが1つ以上あり、数字も具体的 |
| 🟡 要改善 | ケーススタディはあるが、詳細や数字が不足 |
| 🔴 要追加 | ケーススタディがない |

### 3. 図解・挿絵 🎨

概念やプロセスが視覚的に表現されているかを検証します。

**チェックポイント**:
- [ ] 概念図やフロー図が適切に配置されているか
- [ ] キャラクター付き挿絵（シーン説明用）があるか
- [ ] データや比較にはシンプルな図解（棒グラフ、比較表など）があるか
- [ ] 画像参照（`![alt](./xxx.png)`）が本文中に存在するか

**評価基準**:

| 評価 | 基準 |
| --- | --- |
| 🟢 優秀 | 3枚以上の図解/挿絵があり、内容と合致している |
| 🟡 要改善 | 1〜2枚の図解があるが、追加で視覚化すべき箇所がある |
| 🔴 要追加 | 図解がない、または本文とマッチしていない |

### 4. 理論的根拠 📚

主張に対する科学的・論理的な説明があるかを検証します。

**チェックポイント**:
- [ ] 主張に対する科学的・論理的な説明があるか
- [ ] 「Lost in the Middle」などの研究への言及があるか（該当する場合）
- [ ] 「なぜそうなるのか」のメカニズム説明があるか
- [ ] 数字やデータに出典があるか

**評価基準**:

| 評価 | 基準 |
| --- | --- |
| 🟢 優秀 | 主張の裏付けが明確で、必要な研究引用もある |
| 🟡 要改善 | 裏付けはあるが、一部の主張に根拠が不足 |
| 🔴 要追加 | 主張に対する根拠がほとんどない |

### 5. 章間の一貫性 ⚖️

前後の章と比較して同等の詳細レベルかを検証します。

**チェックポイント**:
- [ ] 前の章と同等レベルの詳細さがあるか
- [ ] 用語や説明スタイルが統一されているか
- [ ] 章の分量が大きく異なっていないか（±30%以内が目安）
- [ ] 各章で同等数の具体例・図解があるか

**評価基準**:

| 評価 | 基準 |
| --- | --- |
| 🟢 優秀 | 前後の章と同等の充実度 |
| 🟡 要改善 | 一部の観点で前後の章と差がある |
| 🔴 要追加 | 明らかに他の章より薄い |

## 実行手順

1. **対象ファイルを読み込む**
   - 指定された章ファイルの内容を取得

2. **比較対象の章を読み込む**（章間一貫性チェック用）
   - 同じ部の他の章ファイルを取得
   - 特に前編・後編の関係がある場合は必ず比較

3. **各観点をチェック**
   - 5つのチェック項目について順次評価
   - 各項目で具体的な問題箇所を特定

4. **スコアと改善案を出力**

## 使用例

```bash
# 単一ファイルの充実度チェック
/proofread-depth books/ai-small-is-accurate/part1_why-ai-fails/01b_hidden-costs-and-solutions.md

# 前編と後編を比較してチェック
/proofread-depth books/ai-small-is-accurate/part1_why-ai-fails/01b_hidden-costs-and-solutions.md compare:01_the-seventy-percent-problem.md
```

## 関連スキル

- `/proofread` - 総合校正（日本語・構造・用語・Markdown・ファクトチェック）
- `/proofread-structure` - 章構造のみを検証
- `/proofread-facts` - ファクトチェックのみを実行

## 注意事項

- このスキルは「構造」ではなく「内容の深さ」を評価します
- 構造の問題（必須セクションの欠落など）は `/proofread-structure` で検出します
- 日本語の誤りは `/proofread` で検出します
- 本スキルは `books/ai-small-is-accurate/` 専用です
