# 校正スキル（/proofread）

## 概要

日本語で書かれた技術書の校正を行います。以下の6つの観点で検査し、重要度別にレポートを出力します。

## 検査観点

1. **日本語表現** - 誤字脱字、文法、読みやすさ
2. **章構造** - フォーマット準拠、必須セクションの有無
3. **用語統一** - 表記揺れ、略語ルール、固有名詞
4. **Markdown形式** - コードブロック、リンク、テーブル
5. **ファクトチェック** - 研究論文、統計データ、技術的主張の裏付け
6. **用語抽出**（初回のみ）- 本文から用語集を自動生成

## 使用方法

### 基本的な使い方

```text
/proofread [ファイルパス]
```

### 例

```bash
# 単一ファイルの校正
/proofread books/ai-small-is-accurate/part1_why-ai-fails/01_the-seventy-percent-problem.md

# ディレクトリ全体の校正
/proofread books/ai-small-is-accurate/

# 特定の観点のみ実行
/proofread books/ai-small-is-accurate/ focus:facts
/proofread books/ai-small-is-accurate/ focus:structure
/proofread books/ai-small-is-accurate/ focus:terms
```

### オプション

| オプション | 説明 |
| --- | --- |
| `focus:japanese` | 日本語校正のみ実行 |
| `focus:structure` | 章構造検証のみ実行 |
| `focus:terms` | 用語統一チェックのみ実行 |
| `focus:markdown` | Markdown検証のみ実行 |
| `focus:facts` | ファクトチェックのみ実行 |
| `quick` | 構造とMarkdownのみの高速チェック |
| `full` | 全観点の詳細チェック（デフォルト） |

## 実行手順

1. **対象ファイルの特定**
   - 指定されたパスがファイルかディレクトリかを判定
   - ディレクトリの場合は `.md` ファイルを再帰的に収集

2. **用語集の確認**
   - `terminology.yaml` が存在しない場合は `/extract-terms` を先に実行を推奨
   - 存在する場合はそれを使用

3. **各観点の検査を実行**
   - 日本語校正 → 構造検証 → 用語統一 → Markdown検証 → ファクトチェック
   - 各検査で問題を検出し、重要度を分類

4. **レポート出力**
   - 検出された問題を重要度別に整理
   - サマリーと詳細を出力

## 重要度の基準

| 重要度 | 記号 | 基準 | 例 |
| --- | --- | --- | --- |
| 重大 | 🔴 | 内容の誤り、読者の誤解を招く | 誤字による意味変化、壊れたリンク、出典なしの統計 |
| 警告 | 🟡 | 品質低下、一貫性の欠如 | 用語の不統一、構造の逸脱、古い情報 |
| 提案 | 🔵 | より良くできる点 | 表現の改善、冗長な記述 |

## 出力形式

```markdown
## 校正結果レポート

### サマリー
- 対象ファイル: [ファイル名]
- 検出数: 🔴 重大: X件 / 🟡 警告: Y件 / 🔵 提案: Z件

---

### 🔴 重大な問題（修正必須）

1. **[行番号]** [カテゴリ]: [問題の説明]
   - 現在: 「[現在の記述]」
   - 推奨: 「[推奨する修正]」
   - 理由: [修正が必要な理由]

---

### 🟡 警告（修正推奨）

1. **[行番号]** [カテゴリ]: [問題の説明]
   ...

---

### 🔵 改善提案

1. **[行番号]** [カテゴリ]: [提案内容]
   ...
```

## 関連スキル

- `/proofread-structure` - 章構造のみを検証
- `/proofread-facts` - ファクトチェックのみを実行
