# 主張校閲スキル

## 役割

文章内の主張を検証し、根拠の有無、断言の妥当性、過度な一般化を検出します。

**対象**: 技術書、ビジネス文書、論説文

## 検証対象

### 1. 根拠のない主張

#### 検出パターン

| パターン | 例 | 問題点 |
| --- | --- | --- |
| 無根拠の断言 | 「これが最善である」 | 比較・根拠なし |
| 伝聞の不明確 | 「〜と言われている」 | 出典なし |
| 権威への訴え | 「専門家によると」 | 誰かが不明 |
| 多数への訴え | 「多くの人が」 | 具体的な数値なし |

#### 重要度分類

**🔴 重大（修正必須）**

```
パターン: 事実として断言（根拠なし）
例: 「AIは人間を超えた」
問題: 客観的根拠なしに事実として提示
推奨: 根拠を追加、または意見であることを明示
```

**🟡 警告（根拠追加推奨）**

```
パターン: 曖昧な伝聞
例: 「一般的に〜と考えられている」
問題: 出典や根拠が不明確
推奨: 具体的な出典を追加、または表現を和らげる
```

**🔵 提案（表現の改善）**

```
パターン: 過度に強い表現
例: 「絶対に〜すべき」
問題: 読者に押し付けがましい印象
推奨: 「〜することを推奨」などに和らげる
```

### 2. 過度な断言

#### 危険な断言表現

| 表現 | リスク | 推奨代替 |
| --- | --- | --- |
| 「絶対に」 | 例外を否定 | 「多くの場合」 |
| 「必ず」 | 100%保証 | 「ほぼ確実に」 |
| 「常に」 | 例外なし | 「一般的に」 |
| 「誰でも」 | 全員に適用 | 「多くの人が」 |
| 「唯一の」 | 他の選択肢を否定 | 「最も効果的な」 |
| 「間違いなく」 | 確実性の過大評価 | 「おそらく」 |

#### 検証方法

1. 断言表現を抽出
2. 根拠の有無を確認
3. 例外が存在しないか検討
4. 適切な表現への修正を提案

### 3. 過度な一般化

#### 検出パターン

| パターン | 例 | 問題点 |
| --- | --- | --- |
| 経験からの一般化 | 「私はこうだった→皆そうだ」 | 個人の経験を一般化 |
| 少数事例の一般化 | 「A社で成功→全社で成功」 | サンプル不足 |
| ステレオタイプ | 「エンジニアは〜」 | 多様性の無視 |
| 時代の一般化 | 「昔は〜だった」 | 過去の単純化 |

#### 検証方法

```
一般化のチェックリスト:
□ 主張の範囲は適切か（「すべて」→「多くの」）
□ 根拠となる事例は十分か
□ 例外は考慮されているか
□ 対象グループは適切に限定されているか
```

### 4. 主観と客観の混同

#### 検証項目

| 項目 | 主観的表現 | 客観的表現 |
| --- | --- | --- |
| 評価 | 「素晴らしい」 | 「評価が高い」+出典 |
| 比較 | 「最高の」 | 「〜の点で優れた」 |
| 感情 | 「誰もが喜ぶ」 | 「多くのユーザーが好評」+出典 |

#### 検証方法

1. 評価表現を抽出
2. 主観か客観か判定
3. 客観的根拠の有無を確認
4. 必要に応じて出典追加を推奨

### 5. 断言の妥当性スケール

主張を以下のスケールで評価:

```
[確実] ←――――――――――――→ [不確実]

✅ 確実（断言可）
   - 数学的事実
   - 検証済みの科学的事実
   - 公式ドキュメントに基づく仕様

🟡 高確度（条件付き断言可）
   - 査読済み研究の結論
   - 広く認められた業界標準
   - 複数の信頼できる情報源で一致

🟠 中確度（意見として表明）
   - 経験則、ベストプラクティス
   - 単一の研究結果
   - 専門家の見解

⚠️ 低確度（推測として明示）
   - 個人の経験
   - 少数の事例
   - 予測、推測
```

## 検証プロセス

### ステップ1: 主張の抽出

テキストから以下を抽出:

1. 断言: 「〜である」「〜だ」
2. 評価: 「良い」「悪い」「最高」「最悪」
3. 推奨: 「〜すべき」「〜が良い」
4. 予測: 「〜になる」「〜だろう」

### ステップ2: 根拠の確認

各主張について:
1. 根拠が明示されているか
2. 根拠の種類（データ、研究、経験、推測）
3. 根拠の信頼性

### ステップ3: 妥当性の評価

断言の妥当性スケールに基づいて評価:
- 断言として適切か
- 表現を和らげるべきか
- 根拠を追加すべきか

## 出力形式

```markdown
### 主張校閲結果

#### サマリー
- 検証した主張: X件
- 🔴 根拠必須: X件
- 🟡 根拠推奨: X件
- 🔵 表現改善: X件
- ✅ 問題なし: X件

---

#### 🔴 根拠のない断言（修正必須）

1. **行78**: 事実として断言
   - 主張: 「AIエージェントは人間より優れている」
   - 問題: 根拠なしに事実として提示
   - 推奨A: 根拠を追加「〜の研究によると...」
   - 推奨B: 意見として明示「筆者の経験では...」

2. **行145**: 無根拠の比較
   - 主張: 「これが最も効率的な方法だ」
   - 問題: 他の方法との比較根拠がない
   - 推奨: 「〜という点で効率的な方法の一つだ」

---

#### 🟡 根拠追加推奨

1. **行203**: 曖昧な伝聞
   - 主張: 「多くのエンジニアがこの問題に直面している」
   - 問題: 「多く」の定義が不明確
   - 推奨: 調査データがあれば追加、なければ「筆者の観察では」

2. **行267**: 権威への訴え
   - 主張: 「専門家によると〜」
   - 問題: 専門家が誰か不明
   - 推奨: 具体的な名前・出典を明記

---

#### 🔵 表現の改善提案

1. **行45**: 過度な断言
   - 現在: 「絶対にこの方法を使うべきだ」
   - 推奨: 「この方法を強く推奨する」

2. **行312**: 全称表現
   - 現在: 「すべてのプロジェクトで効果がある」
   - 推奨: 「多くのプロジェクトで効果が期待できる」

---

#### ✅ 適切な主張

1. **行180**: 「本書では〜というアプローチを提案する」
   - 筆者の意見として適切に表明
   - 押し付けがましくない表現
```

## チェックリスト

検証時に確認すべき項目:

- [ ] 「〜と言われている」に出典があるか
- [ ] 「最高」「最善」などの評価に根拠があるか
- [ ] 「常に」「絶対に」「必ず」に例外の考慮があるか
- [ ] 「多くの」「ほとんどの」に具体的なデータがあるか
- [ ] 個人の経験を一般化していないか
- [ ] 主観的評価と客観的事実が区別されているか
- [ ] 断言の強さと根拠の確かさが釣り合っているか
