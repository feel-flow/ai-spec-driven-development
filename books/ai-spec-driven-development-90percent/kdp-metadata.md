## 内容紹介

【現役CTOが教える、AIエージェント時代の開発バイブル】

Claude Code、GitHub Copilotなどの AIコーディングツールを「なんとなく」使っていませんか？

本書は、AI仕様駆動開発（AI Spec-Driven Development）という新しいアプローチで、
AIエージェントに安心して開発を任せるための「設計図」を提供します。

■ こんな方におすすめ
・AIコーディングツールを導入したが、期待ほど成果が出ない
・Vibe Codingで何度もやり直しが発生している
・AIに任せる範囲と人間が判断すべき範囲の線引きがわからない

■ 本書の特徴
・たった7文書で開発を制御する「仕様駆動」メソッド
・Claude Code / GitHub Copilot 両対応の実践テクニック
・現場で即使えるテンプレート・設定ファイル付き
・★特典：Skills作成ガイド付き

■ 構成
Part 1：なぜAI開発は失敗するのか
Part 2：仕様が9割 ─ 7文書構成の威力
Part 3：実践ワークフロー
Part 4：FAQ ─ エンジニアの役割・品質・セキュリティ
Part 5：チーム展開と組織戦略

AIに「何を作るか」を正しく伝える技術。それがAI仕様駆動開発です。
