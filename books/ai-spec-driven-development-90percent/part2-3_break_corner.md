## 💡 7文書で失敗しないための実践Tips

【DJ町娘】「AI侍さん、7文書を使い始める時、よくある失敗ってありますか？」

【AI侍】「うむ、良い質問だ。実践で得た3つのTipsを授けよう」

### Tip 1：完璧を目指すな、育てよ

【AI侍】「初心者が最もよくやる失敗——それは『完璧な文書を最初から作ろうとすること』だ」

❌ **失敗パターン**：
- MASTER.mdに全ての情報を詰め込もうとする
- 文書が完成するまでAIに指示を出さない
- 1週間かけて文書を書いたが、結局使われない

⭕ **成功パターン**：
- MASTER.mdは最初30行でOK（プロジェクト名、技術スタック、ディレクトリ構造だけ）
- すぐにAIに指示を出して試す
- レビューで指摘されたら、その場で文書に追記

【DJ町娘】「なるほど！まずは小さく始めて、育てていくんですね」

### Tip 2：「正」の在り処を明確にせよ

【AI侍】「同じ情報を複数の文書に書く——これも典型的な失敗だ」

❌ **失敗例**：
- 認証方式がARCHITECTURE.mdとPATTERNS.md両方に書かれている
- 内容が微妙に違う
- どちらが正しいか分からなくなる

⭕ **解決策**：

```markdown
## ARCHITECTURE.md（What: 何を使うか）
- 認証方式: JWT (HS256)
- 有効期限: 24時間

## PATTERNS.md（How: どう書くか）
- JWTの生成/検証は src/auth/jwt.ts を使用
- トークンは Authorization: Bearer {token} ヘッダーで送信
```

【AI侍】「**What（何を）はARCHITECTURE、How（どう）はPATTERNS**——この区別を守れ」

### Tip 3：「迷ったら MASTER」を合言葉に

【DJ町娘】「でも、どの情報をどの文書に書けばいいか迷っちゃいます...」

【AI侍】「その時は**まずMASTER.mdに書け**。迷っている時間が一番無駄だ」

```markdown
## MASTER.md（一時的なメモとして）
### TODO: 整理が必要な情報
- ユーザー権限の仕様（→ 後でDOMAIN.mdに移動）
- API エラーコード一覧（→ 後でPATTERNS.mdに移動）
```

【AI侍】「MASTER.mdに一旦メモして、後で適切な場所に移せばよい。立ち止まるな、書き続けよ」

【DJ町娘】「わかりました！まず書いて、後で整理する✨」
